# G-02 Template Management

> **Phase:** MVP 1 | **Status:** In Progress
> **Specs:** [API &sect;5.3, &sect;5.4, &sect;5.6](../specs/5-api-design.md) &middot; [Data](../specs/6-data-model.md)

## Goal Statement

使用者可在後台選取 template，並完成建立、修改、刪除與模板動作管理，作為開始訓練的主要入口。

## Requirement

系統必須提供 template CRUD、template item CRUD，以及可支援後續開始訓練流程的穩定模板編輯體驗。

## In Scope

1. Template CRUD
2. Template item CRUD（新增、修改、刪除、排序）
3. Template list 與 editor UI
4. 從 template 選取並作為開始訓練入口

## Out Scope

1. Template 分享
2. 通知中心
3. 唯讀預覽與 copy-from-share
4. 跨版本自動衝突解決

## Frontend Elements

1. Template list page（可選取 template）
2. Template create / edit page
3. Template item list（動作、組數、預設欄位）
4. Template 刪除確認流程
5. 後台模板動作新增 / 修改 / 刪除 UI

## Acceptance Criteria

- [ ] Given 使用者進入模板列表，When 查看既有資料，Then 可清楚選取任一 template 進入編輯或開始訓練。
- [ ] Given 使用者新增、移動或刪除 template item，When 儲存成功，Then template item 順序由 server 端維護，並以 transaction 保證同一份 template 內的 `sortOrder` 一致且唯一。
- [ ] Given 使用者建立新的 template，When 儲存成功，Then 該 template 會出現在模板列表並可立即用於開始訓練。
- [ ] Given 使用者修改 template 名稱或其中動作，When 儲存成功，Then 重新進入時可看到最新內容。
- [ ] Given 使用者刪除 template，When 確認刪除成功，Then 該 template 不再出現在模板列表且不可再被選取。
- [ ] Given 使用者在模板內建立、編輯或刪除動作，When 儲存成功，Then template 內容與後續開始訓練頁資料一致。
- [ ] Given 使用者建立自訂動作，When 建立成功，Then 該動作可加入 template 並被保存。
