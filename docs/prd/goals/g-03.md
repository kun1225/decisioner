# G-03 Workout Session Flow

> **Phase:** MVP 1 | **Status:** Not Started
> **Specs:** [API &sect;5.6, &sect;5.7](../specs/5-api-design.md) &middot; [Data](../specs/6-data-model.md)

## Goal Statement

使用者可從 template 開始一場訓練，快速輸入各動作的重量與次數，並順利完成訓練。

## Requirement

系統必須提供從 template 啟動 session、預設 3 組輸入列、可加組，以及清楚的訓練完成流程。

## In Scope

1. 由 template 啟動 session
2. 依 template 列出動作與 set 輸入列
3. 每個動作預設顯示 3 組
4. 可用 `+` 新增第 4 組以上
5. 完成訓練按鈕與完成後彩帶回饋

## Out Scope

1. Session 內替換/刪除/新增動作的進階編輯器
2. 調整自動回寫 template
3. 跨 session 批次套用調整

## Frontend Elements

1. 開始訓練頁 template 選擇器
2. 動作清單與 set 輸入列（weight/reps/unit）
3. 每個動作的 `+` add set button
4. 頁面底部固定訓練完成按鈕
5. 完成後彩帶 / success feedback

## Acceptance Criteria

- [ ] Given 使用者選擇 template 開始訓練，When 進入訓練頁，Then 系統需列出該 template 的所有動作與對應輸入列。
- [ ] Given 每個動作進入訓練頁，When 頁面初次載入，Then 預設顯示 3 組可輸入的 `weight/reps/unit`。
- [ ] Given 使用者點擊某動作底部的 `+`，When 新增成功，Then 該動作可繼續輸入第 4 組以上資料。
- [ ] Given 使用者在 session 記錄 sets，When 儲存 set 資料，Then 每組至少包含 `weight/reps/unit/set_index`。
- [ ] Given 使用者完成訓練，When 點擊頁面最下方的完成按鈕，Then session 狀態更新為 completed 並顯示彩帶回饋。

## Tasks

- [ ] Start session from template（snapshot 建立 API + DB）
- [ ] Set CRUD API + DB
- [ ] Session 前端訓練頁面（template 選擇 + 動作清單 + set 輸入）
- [ ] 預設 3 組與 add set UI 邏輯
- [ ] 訓練完成按鈕與彩帶互動
