# G-07 Friend Feed Like

> **Phase:** 1.0 | **Status:** Not Started
> **Specs:** [API &sect;5.9](../specs/5-api-design.md) &middot; [Data](../specs/6-data-model.md)

## Goal Statement

使用者可在好友動態看到訓練/打卡事件並進行愛心互動。

## Requirement

系統必須提供好友動態 feed 與 like/unlike 行為。

## In Scope

1. Feed 事件：`WORKOUT_STARTED`、`WORKOUT_COMPLETED`、`CHECKIN_CREATED`
2. Like / Unlike
3. 基本個人頁入口（從 feed 點進）

## Out Scope

1. 留言與回覆
2. 通知中心

## Frontend Elements

1. 好友動態 feed 清單項目
2. 愛心按鈕與計數顯示
3. 事件類型標籤（開始訓練/完成訓練/打卡）
4. 從 feed 進入的基本個人頁入口

## Acceptance Criteria

- [ ] Given 好友產生事件，When 使用者讀取 feed，Then 可看到該事件。
- [ ] Given 使用者對事件按愛心，When 同事件重複按愛心，Then 不會重複計數。
- [ ] Given 使用者從 feed 點擊好友頭像，When 進入 profile，Then 可看到受隱私控制後的個人頁內容。

## Tasks

- [ ] Activity feed event 寫入（workout/checkin 觸發）
- [ ] Feed query API（好友範圍 + 分頁）
- [ ] Like/Unlike API + DB
- [ ] Feed 前端頁面（事件清單 + 愛心互動）
