# G-20 Page-oriented Frontend Structure

> **Phase:** 1.0 | **Status:** Not Started
> **Specs:** [Frontend &sect;4.1, &sect;4.3](../specs/4-frontend-architecture.md)

## Goal Statement

前端專案以頁面為主體切分，讓每個 route 的 UI 與業務邏輯可在同一目錄內維護。

## Requirement

系統必須採用 route-first 檔案架構，頁面私有邏輯放在 route 目錄下，跨頁共用邏輯集中於 `src/lib`，providers 由單一入口匯出。

## In Scope

1. `src/routes/<page>` 底下維護頁面私有檔案
2. 每個 route 可使用 `/_components` 與 `/_domain`
3. `src/providers/index.tsx` 匯出單一 `Provider` 元件
4. 跨頁共用的 feature/domain 模組集中到 `src/lib`
5. auth 路由結構範例（`login.tsx`、`register.tsx`）

## Out Scope

1. 以 `features/*` 作為主切分
2. 以全域 `domain/*` 作為主切分
3. 本階段進行實際功能重寫或搬遷

## Frontend Elements

1. `/src/routes` page-oriented 目錄結構
2. `/src/routes/auth/_components` 與 `/src/routes/auth/_domain`
3. `/src/providers/auth-gate.tsx`
4. `/src/providers/index.tsx`（single provider export）
5. `/src/lib` 全域共用模組目錄

## Acceptance Criteria

- [ ] Given 新增 auth 頁面，When 建立檔案，Then `login.tsx` 與 `register.tsx` 位於 `src/routes/auth`。
- [ ] Given 某頁面有私有元件或私有邏輯，When 實作時，Then 優先放在該 route 目錄下的 `/_components` 或 `/_domain`。
- [ ] Given 模組需被多個頁面共用，When 抽離時，Then 放入 `src/lib` 而非建立全域 `features/*` 或 `domain/*`。
- [ ] Given App 需要掛載 providers，When 匯入 provider，Then 只需從 `src/providers/index.tsx` 匯入單一 `Provider` 元件。

## Tasks

- [ ] Route-first 目錄結構建立（src/routes + _components + _domain）
- [ ] Providers 單一入口匯出（src/providers/index.tsx）
- [ ] Auth 路由結構範例（login + register）
- [ ] src/lib 共用模組目錄建立
